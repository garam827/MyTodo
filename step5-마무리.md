# MyTodo — Step 5: 접근성 · 크로스 브라우저 · 최종 점검

> 전제: Step 4 완료 (모든 기능 구현됨)

MyTodo 앱의 마무리 작업을 해줘.

## 1. 키보드 접근성
모든 기능이 키보드만으로 사용 가능해야 함:
- Tab: 모든 인터랙티브 요소 (버튼, 입력, 체크박스) 순서대로 이동
- Space: 체크박스 토글
- Enter / Space: 버튼 실행
- 좌우 화살표: 필터 탭 전환 (선택사항)
- focus-visible 스타일: 포커스 링 명확하게 표시

## 2. 반응형 레이아웃 점검
다음 화면 너비에서 레이아웃 깨짐 없어야 함:
- 320px: 모바일 세로 (최소)
- 375px: iPhone SE
- 768px: 태블릿
- 1024px+: 데스크탑

필터 탭이 좁은 화면에서 가로 스크롤 없이 표시되어야 함
(flex-wrap 또는 overflow-x: auto 처리)

## 3. 엣지케이스 처리
- localStorage 용량 초과 시: try-catch로 에러 캐치 후 콘솔 경고
- 매우 긴 텍스트 입력 시: text-overflow: ellipsis로 잘림 처리
  (편집 모드에서는 전체 텍스트 표시)
- XSS 방지: innerHTML 대신 textContent 사용
- 항목 20개 이상: 목록이 길어져도 레이아웃 정상 유지

## 4. 최종 검증 체크리스트
다음 시나리오를 실제로 테스트하고 문제가 있으면 수정:
- [ ] 할 일 추가 → 새로고침 → 데이터 유지 확인
- [ ] 완료 체크 → 새로고침 → 완료 상태 유지 확인
- [ ] 카테고리 필터 탭 전환 → 올바른 항목만 표시
- [ ] 인라인 수정 → Escape 취소 → 원본 유지 확인
- [ ] 빈 입력 추가 시도 → 차단 확인
- [ ] 완료 항목 일괄 삭제 → 미완료 항목 보존 확인
- [ ] 모바일 화면(320px)에서 레이아웃 정상

## 5. 다크 테마
Step 1에서 정의한 CSS 변수 시스템을 활용해 다크 테마를 추가:

헤더 우측에 토글 버튼 추가:
```html
<button id="btn-theme" aria-label="테마 전환">🌙</button>
```

동작:
- 클릭 시 `<body>`에 `data-theme="dark"` 속성 토글
- 라이트 모드 아이콘: 🌙 / 다크 모드 아이콘: ☀️
- 선택한 테마를 localStorage(`mytodo_theme`)에 저장 → 새로고침 후 유지
- 시스템 다크모드(`prefers-color-scheme: dark`) 자동 감지하여 초기값 적용

CSS 변수 오버라이드 (`[data-theme="dark"]`):
- `--color-bg`: #0f1117
- `--color-surface`: #1a1d27
- `--color-border`: #2d3148
- `--color-text`: #e2e8f0
- `--color-text-muted`: #94a3b8
- 카테고리 배지 색상도 어두운 톤으로 조정
